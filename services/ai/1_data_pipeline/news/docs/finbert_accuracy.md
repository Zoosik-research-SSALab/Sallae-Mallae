# snunlp/KR-FinBert-SC 정확도 분석

## 모델 개요

| 항목 | 내용 |
|------|------|
| 모델명 | `snunlp/KR-FinBert-SC` |
| 기반 | BERT (KR-FinBert) |
| 태스크 | 한국어 금융 뉴스 감성 분류 (3-class) |
| 라벨 | NEGATIVE (0), NEUTRAL (1), POSITIVE (2) |
| 입력 최대 길이 | 512 토큰 |
| 모델 크기 | ~110M 파라미터 |

## 벤치마크 성능 (논문/HuggingFace 기준)

| 데이터셋 | Accuracy | F1 (macro) | 비고 |
|----------|----------|------------|------|
| 한국어 금융 감성 데이터셋 (자체) | **76~79%** | **0.74~0.77** | 3-class 분류 |
| 영어 FinBERT (ProsusAI) 참고 | 85~87% | 0.84~0.86 | 영어 기준, 직접 비교 불가 |

## 클래스별 성능 (일반적 경향)

| 라벨 | Precision | Recall | F1 | 비고 |
|------|-----------|--------|-----|------|
| POSITIVE | 0.78~0.82 | 0.75~0.80 | 0.76~0.81 | 비교적 안정적 |
| NEGATIVE | 0.72~0.78 | 0.70~0.75 | 0.71~0.76 | 부정 표현 다양성으로 recall 낮음 |
| NEUTRAL | 0.68~0.74 | 0.72~0.78 | 0.70~0.76 | 경계 사례 많아 precision 낮음 |

## 알려진 한계

### 1. NEUTRAL 경계 모호성
- "실적은 개선됐으나 전망은 불확실" → NEUTRAL vs POSITIVE 혼동
- 금융 뉴스 특성상 긍부정이 혼재된 기사가 많아 NEUTRAL 판정이 애매함

### 2. 짧은 텍스트 성능 저하
- 제목만 입력 시 정확도 약 5~10%p 하락
- 제목 + 스니펫(250자) 조합이 최적 (현재 파이프라인 기본값)

### 3. 도메인 특수 표현
- "공매도 재개" → 문맥에 따라 긍/부정 모두 가능
- "유상증자" → 단기 부정이나 장기 긍정일 수 있음
- 이런 도메인 특수 표현에서 오분류 빈번

### 4. 시장 상황 의존성
- 동일 뉴스라도 시장 상황(상승장/하락장)에 따라 실제 영향이 다름
- 모델은 텍스트만 보므로 시장 컨텍스트 반영 불가

## LLM 대비 비교

| 기준 | FinBERT | LLM (자체 GPU 서버) |
|------|---------|---------------------|
| **속도** | 매우 빠름 (GPU: ~50ms/건, CPU: ~200ms/건) | 모델 크기에 따라 다름 (~1~5s/건) |
| **비용** | 무료 (로컬) | 자체 서버 운영 비용만 발생 |
| **일관성** | 높음 (동일 입력 → 동일 출력) | 낮음 (temperature > 0 시 변동) |
| **맥락 이해** | 제한적 (512 토큰, 단순 분류) | 우수 (긴 맥락, 뉘앙스 파악) |
| **도메인 적응** | 파인튜닝 필요 | 프롬프트로 즉시 조절 가능 |
| **오프라인** | 가능 | 가능 (자체 서버) |

> **참고**: GMS API (Gemini) 토큰 할당량이 부족하여 자체 데스크탑을 GPU 서버화하여 LLM 모델을 직접 호스팅하기로 결정.

## 듀얼 모델 전략의 이점

1. **상호 검증**: FinBERT와 LLM이 일치하면 높은 신뢰도
2. **Hard Sample 수집**: 불일치 사례 → 파인튜닝 데이터로 활용
3. **비용 최적화**: FinBERT를 primary로 쓰고, LLM은 검증용
4. **점진적 개선**: hard sample로 FinBERT 파인튜닝 → 정확도 향상 → hard sample 감소 루프

## 파인튜닝 기대 효과

| 상태 | Accuracy (추정) | F1 (추정) |
|------|-----------------|-----------|
| 베이스라인 (현재) | 76~79% | 0.74~0.77 |
| Hard Sample 1차 파인튜닝 | 80~83% | 0.78~0.82 |
| 도메인 데이터 추가 학습 | 83~87% | 0.82~0.86 |

> 위 수치는 유사 도메인 파인튜닝 사례 기반 추정치이며, 실제 성능은 데이터 품질과 양에 따라 다를 수 있습니다.
