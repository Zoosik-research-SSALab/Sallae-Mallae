# Jira convention

### 🔹1. 통합 규칙

- FE/BE가 나뉘는 기능은 반드시 "통합 완료" Story 생성
- 통합 완료 시 기능 완성으로 간주
- 공동 작업이라도 책임자 1명 지정
    - 참여자는 description에 명시
- 스토리 크기 기준
    - 1~3일 내 완료 가능해야 함
    - 3일 초과 예상 시 분리
    - 기능 단위로 설명 가능해야 함

---

### 🔹2. 스프린트

- 스프린트는 최소 1주 단위로 운영한다.
- 스프린트 시작 전, 백로그에 Story를 모두 추가한다.
- 스프린트 목표는 Description에 명시한다.
    - 예: “1차 MVP 기능 구현 완료”
- 스크럼 회의 시 Jira 보드를 기준으로 진행한다.
- Story가 완료 기준을 충족해야 Done 처리한다.

---

### 🔹3. 스토리(스크럼) 포인트

- 사용 포인트
    - 1 2 3 5 8만 사용
    - 8pt 초과 금지
- 기준
    - 1명당 주당 15~21pt     (조정 가능)
    - 6명 기준 90~126pt / 주
    - Velocity는 일정 예측 참고용
    - 개인 pt는 참고용 (성과 평가 기준 아님)
- 원칙
    - 포인트는 “합의”로 정한다.
    - 빠르게 찍지 말고 1~2분 토론 후 결정.
    - 리스크가 크면 5 대신 8로 올린다.
    - 애매하면 큰 값 선택

### 🔹4.

### EPIC → 기능 영역 정의

### 📌 제목 형식

```
인증 및 회원 시스템
```

### 📌 Description 구조

```
목적:
이 Epic이 해결하는 큰 기능 영역

범위:
포함되는 주요 기능 나열

완료 기준:
이 Epic이 끝났다고 판단하는 상태
```

---

### 📌 예시

```
목적:
사용자가 회원가입 및 로그인하여 인증이 필요한 기능을 이용할 수 있도록 한다.

범위:
- 회원가입
- 로그인
- 아이디/비밀번호 찾기
- JWT 토큰 발급 및 검증
- 인증이 필요한 리포트 접근 제어

완료 기준:
- 회원가입/로그인 정상 동작
- 로그인 없이 리포트 접근 불가
- 로그아웃 시 토큰 무효화
```

### STORY → 기능 단위

허용 가능 태그

| 태그 | 영역 | 해당 작업 예시 |
| --- | --- | --- |
| `[BE]` | 백엔드 | API 구현, 서버 로직, DB 설계, 인증/인가 |
| `[FE]` | 프론트엔드 | UI 화면, 컴포넌트, 인터랙션, 스타일 |
| `[DATA]` | 데이터 수집·파이프라인 | 수집기 구현, ETL, 데이터 파싱, 저장 |
| `[AI]` | AI·ML·피처·모델 | 피처 엔지니어링, 모델 학습, 추론, 평가 |
| `[INFRA]` | 인프라·운영 | 스케줄러, 배포, CI/CD, 환경 설정, 모니터링 |
| `[DESIGN]`  | 디자인 | 와이어프레임, 디자인 |

## 복합 태그

두 영역에 걸친 작업은 `/` 로 구분합니다.

```
[STORY] [BE/FE] 로그인 화면 및 API 연동
[STORY] [DATA/ML] 수급 데이터 기반 피처 생성 파이프라인
```

---

## 태그 선택 기준

애매한 경우 아래 기준으로 결정합니다.

- **데이터를 수집·저장하는 작업** → `[DATA]`
- **수집된 데이터를 가공·변환하는 작업** → `[DATA]` 또는 `[AI]`
    - 범용 전처리(정규화, 결측치 처리) → `[DATA]`
    - 모델 학습용 피처 생성 → `[AI]`
- **모델 학습·추론·평가** → `[AI]`
- **AI를 포함한 서비스 로직** → `[BE]` (API) + `[AI]` (모델) 로 분리
- **배포·자동화·환경 설정** → `[INFRA]`

### 📌 제목 형식

```

```

---

### 📌 Description 구조

```
문제:
어디서 어떤 오류 발생

정상 동작:
원래 어떻게 동작해야 하는지

재현 방법:
간단 단계
```

---

### 📌 예시

```
문제:

정상 동작:
리포트 생성 후 분석 결과 반환

재현 방법:
1. 로그인
2. 종목 상세 페이지 진입
3. 리포트 생성 클릭
```
