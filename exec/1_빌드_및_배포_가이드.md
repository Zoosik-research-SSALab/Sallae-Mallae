## 1. 프로젝트 개요

종목, 뉴스, 공시, 재무, AI 분석 결과를 한 화면에서 통합 제공하는 주식 추천/분석 서비스입니다.

### 아키텍처

```
[Client Browser]
       │
       ▼
[Gateway Nginx] (:80/:443)  ─── j14d208.p.ssafy.io
       ├── /          → Frontend (Next.js :3000)
       ├── /api/      → Backend  (Spring Boot :8080)
       ├── /ai/       → AI Server (FastAPI :8000)
       └── /assets/   → MinIO Object Storage (:9000)

[Shared Infrastructure]
       ├── PostgreSQL 16 (TimescaleDB + pgvector)
       ├── Redis 7
       └── MinIO (S3-compatible)
```

## 3. 소스 클론 및 빌드

### 3.1 GitLab 소스 클론

```bash
git clone https://lab.ssafy.com/s14-bigdata-recom-sub1/S14P21D208.git
cd S14P21D208
```

### 3.2 Backend 빌드

#### 로컬 실행

```bash
cd services/backend
./mvnw clean package -DskipTests
java -jar target/sallaemallae-backend-0.0.1-SNAPSHOT.jar
```

#### Docker 빌드

```bash
cd services/backend
docker build -t sallaemallae-backend .
docker run -p 8080:8080 \
  -e DB_HOST=<host> -e DB_PASSWORD=<pw> \
  -e REDIS_HOST=<host> -e JWT_SECRET=<secret> \
  sallaemallae-backend
```

> **Dockerfile**: Multi-stage 빌드
> - Stage 1: `maven:3.9.9-eclipse-temurin-21` → Maven 빌드, JAR 생성
> - Stage 2: `eclipse-temurin:21-jre` → JRE만 포함하여 실행

### 3.3 Frontend 빌드

#### 로컬 실행

```bash
cd services/frontend
corepack enable
pnpm install
pnpm dev          # 개발 서버 (port 3000)
```

#### Docker 빌드

```bash
cd services/frontend
docker build -t sallaemallae-frontend .
docker run -p 3000:3000 sallaemallae-frontend
```

> **Dockerfile**: `node:22-alpine` → pnpm install → build → start

### 3.4 AI 서버 빌드

AI 서비스는 4개 파이프라인으로 구성됩니다.

```
services/ai/
├── 1_data_pipeline/stock/    # 주식 데이터 수집 (Python 3.14)
├── 1_data_pipeline/news/     # 뉴스 크롤링/키워드 추출 (Python 3.12)
├── 2_ml_pipeline/            # 모델 학습/추론 (Python 3.14)
├── 3_ai_server/              # FastAPI 추론 서버 (Python 3.12)
└── 4_debate_worker/          # LLM 토론 워커 (Python 3.11+)
```

#### AI Server (FastAPI) 빌드

```bash
cd services/ai/3_ai_server
pip install -r requirements.txt
uvicorn main:app --host 0.0.0.0 --port 8000
```

#### Docker 빌드

```bash
cd services/ai/3_ai_server
docker build -t sallaemallae-ai-server .
docker run -p 8000:8000 sallaemallae-ai-server
```

> **Dockerfile**: `python:3.12-slim` → pip install → uvicorn 실행

#### Stock Data Pipeline

```bash

pip install -r requirements.txt
python pipeline.py --mode incremental    # 데이터 수집
python scheduler.py                       # 스케줄러 (매일 16:30 KST)
```

> **Dockerfile**: `python:3.14-slim` + rclone 설치

#### News Data Pipeline

```bash
cd services/ai/1_data_pipeline/news
pip install -r requirements.txt
python -m crawlers.daily                  # 뉴스 크롤링
python scheduler.py                       # 스케줄러 (매일 16:00 KST)
```

```bash
python scripts/daily_inference.py         # 일일 추론
```

### 5.2 GitLab CI/CD 자동 배포

브랜치 push 시 자동 배포됩니다. 파이프라인 설정은 `.gitlab/infra-deploy.yml`에 정의되어 있습니다.

```
git push → GitLab Runner(ec2) 트리거
  → infra-common 브랜치에서 인프라 설정 다운로드
  → write-env-files.sh로 GitLab Variables에서 env 파일 생성
  → 소스 /srv/sallaemallae/source/<target>으로 동기화
  → docker compose up --build
  → nginx reload
  → Mattermost 알림 전송
```

## 6. 배포 시 특이사항

### 6.1 PostgreSQL 초기화
- 최초 기동 시 `infra/postgres/init/01-init-databases.sh`로 DB 사용자 및 데이터베이스 자동 생성
- `02-init-extensions.sh`로 timescaledb, vector, pg_trgm 확장 설치
- 이후 Backend 기동 시 Flyway가 V1~V22 마이그레이션 자동 실행

### 6.2 SSL 인증서
- 최초 발급: `infra/scripts/issue-gateway-cert.sh`
- 자동 갱신: `infra/scripts/renew-gateway-cert.sh`를 cron에 등록

### 6.3 Gateway Rate Limiting
- API: 20 req/s (burst 60), 동시 연결 40
- Assets: 30 req/s (burst 60), 동시 연결 30
- SSE Stream: 동시 연결 100, 읽기 타임아웃 5분

### 6.4 네트워크 구조
- `sallae-base-net`: 모든 앱이 공유 (PostgreSQL, Redis, MinIO 접근)
- 각 앱별 `app-net`: nginx ↔ 앱 서비스 내부 통신

### 6.5 배포 사전 체크 (스크립트 내장)
- 최소 가용 메모리: 1,536 MB
- 최대 디스크 사용률: 85%
- 최대 CPU load/core: 1.5

### 6.6 SSE (Server-Sent Events)
- `/api/stream/` 경로: proxy_buffering off, chunked_transfer_encoding off

### 6.7 Backend 주요 설정값
- `server.forward-headers-strategy=framework` (리버스 프록시 대응)
- `spring.jackson.property-naming-strategy=SNAKE_CASE` (JSON snake_case)
- `spring.jpa.open-in-view=false` (OSIV 비활성화)

### 6.8 AI 서버 변경 감지 배포
