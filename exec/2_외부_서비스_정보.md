## 1. 소셜 로그인 (OAuth 2.0)

### 1.1 Google OAuth

| 항목 | 내용 |
|------|------|
| 서비스명 | Google Cloud Console - OAuth 2.0 |
| 가입/설정 URL | https://console.cloud.google.com/apis/credentials |
| Token Endpoint | `https://oauth2.googleapis.com/token` |
| User Info Endpoint | `https://www.googleapis.com/oauth2/v2/userinfo` |
| 필요 환경 변수 | `OAUTH_GOOGLE_CLIENT_ID`, `OAUTH_GOOGLE_CLIENT_SECRET`, `OAUTH_GOOGLE_REDIRECT_URI` |

**설정 절차:**
1. Google Cloud Console에서 프로젝트 생성
2. "APIs & Services" → "OAuth 동의 화면" 설정 (외부 사용자 유형)
3. "사용자 인증 정보" → "OAuth 2.0 클라이언트 ID" 생성 (웹 애플리케이션)
4. "승인된 리다이렉트 URI"에 콜백 URL 등록
   - 예: `https://j14d208.p.ssafy.io/api/auth/google/callback`
5. 생성된 Client ID와 Client Secret을 환경 변수에 설정

### 1.2 Naver OAuth (네아로)

| 항목 | 내용 |
|------|------|
| 서비스명 | Naver Developers - 네이버 아이디로 로그인 |
| 가입/설정 URL | https://developers.naver.com/apps/ |
| Token Endpoint | `https://nid.naver.com/oauth2.0/token` |
| User Info Endpoint | `https://openapi.naver.com/v1/nid/me` |
| 필요 환경 변수 | `OAUTH_NAVER_CLIENT_ID`, `OAUTH_NAVER_CLIENT_SECRET`, `OAUTH_NAVER_REDIRECT_URI` |

**설정 절차:**
1. 네이버 개발자센터 (https://developers.naver.com) 가입
2. "Application" → "애플리케이션 등록"
3. 사용 API: "네아로(네이버 아이디로 로그인)" 선택
4. 제공 정보: 회원이름, 이메일 등 필요 항목 선택
5. 서비스 URL 및 Callback URL 등록
   - 예: `https://j14d208.p.ssafy.io/api/auth/naver/callback`
6. 발급된 Client ID와 Client Secret을 환경 변수에 설정

### 1.3 Kakao OAuth (카카오 로그인)

| 항목 | 내용 |
|------|------|
| 서비스명 | Kakao Developers - 카카오 로그인 |
| 가입/설정 URL | https://developers.kakao.com/console/app |
| Token Endpoint | `https://kauth.kakao.com/oauth/token` |
| User Info Endpoint | `https://kapi.kakao.com/v2/user/me` |
| 필요 환경 변수 | `OAUTH_KAKAO_CLIENT_ID`, `OAUTH_KAKAO_CLIENT_SECRET`, `OAUTH_KAKAO_REDIRECT_URI` |

**설정 절차:**
1. 카카오 개발자 사이트 (https://developers.kakao.com) 가입
2. "내 애플리케이션" → 애플리케이션 추가
3. "카카오 로그인" 활성화 (ON)
4. Redirect URI 등록
   - 예: `https://j14d208.p.ssafy.io/api/auth/kakao/callback`
5. "동의항목" 설정 (닉네임, 이메일 등 필수 항목)
6. "앱 키" 탭에서 REST API 키 = Client ID, "보안" 탭에서 Client Secret 발급
7. 환경 변수에 설정

## 2. 한국투자증권 Open API (KIS)

| 항목 | 내용 |
|------|------|
| 서비스명 | 한국투자증권 Open API |
| 가입/설정 URL | https://apiportal.koreainvestment.com/ |
| REST API (실전) | `https://openapi.koreainvestment.com:9443` |
| REST API (모의) | `https://openapivts.koreainvestment.com:29443` |
| WebSocket (실전) | `ws://ops.koreainvestment.com:21000` |
| WebSocket (모의) | `ws://ops.koreainvestment.com:31000` |

**필요 환경 변수:**

| 변수 | 설명 |
|------|------|
| `KIS_MODE` | `paper`(모의투자) / `prod`(실전투자) |
| `KIS_APP_KEY` | API 앱 키 |
| `KIS_APP_SECRET` | API 앱 시크릿 |
| `KIS_ACCOUNT_NO` | 투자 계좌번호 |
| `KIS_ACCOUNT_PRODUCT_CODE` | 계좌 상품코드 |

**설정 절차:**
1. 한국투자증권 계좌 개설 (비대면 가능)
2. Open API 포탈 (https://apiportal.koreainvestment.com) 가입
3. "앱 등록" → App Key, App Secret 발급
4. 모의투자 신청 (테스트용, 포탈에서 가능)
5. 환경 변수에 설정

**주요 API Endpoint:**
- `/oauth2/tokenP`: 접근 토큰 발급
- `/oauth2/Approval`: 승인키 발급 (WebSocket용)
- 실시간 시세, 호가, 체결 데이터 조회

**용도:** 실시간 주가 조회, 호가 데이터, 매매 신호, WebSocket 시세 스트리밍

## 3. 이메일 서비스 (SMTP)

| 항목 | 내용 |
|------|------|
| 기본 Provider | Gmail SMTP |
| Host / Port | `smtp.gmail.com` / 587 (TLS) |
| 필요 환경 변수 | `MAIL_HOST`, `MAIL_PORT`, `MAIL_USERNAME`, `MAIL_PASSWORD` |

**Gmail 설정 절차:**
1. Google 계정 → 보안 → 2단계 인증 활성화
2. https://myaccount.google.com/apppasswords 에서 앱 비밀번호 생성
3. 생성된 앱 비밀번호를 `MAIL_PASSWORD`에 설정

**지원 가능한 대체 Provider:**

| Provider | Host | 비고 |
|----------|------|------|
| AWS SES | `email-smtp.ap-northeast-2.amazonaws.com` | AWS 콘솔에서 설정 |
| SendGrid | `smtp.sendgrid.net` | USERNAME=`apikey` |
| Mailgun | `smtp.mailgun.org` | - |

**용도:** 이메일 인증 코드 발송 (5분 유효), 비밀번호 재설정 코드 발송 (5분 유효)

## 5. MinIO 오브젝트 스토리지 (자체 호스팅)

| 항목 | 내용 |
|------|------|
| 서비스명 | MinIO (S3 호환 오브젝트 스토리지) |
| 배포 방식 | Docker 컨테이너 (Base 인프라에 포함) |
| API Port | 9000 |
| Console Port | 9001 |

**필요 환경 변수:**

| 변수 | 설명 |
|------|------|
| `MINIO_ROOT_USER` | 루트 사용자명 (기본: minioadmin) |
| `MINIO_ROOT_PASSWORD` | 루트 비밀번호 |
| `MINIO_ENDPOINT` | API 엔드포인트 (예: http://minio:9000) |
| `MINIO_ACCESS_KEY` / `SECRET_KEY` | 앱 접근 키 |
| `MINIO_BUCKET` | 사용 버킷명 |
| `MINIO_PUBLIC_URL` | 외부 공개 URL |

**버킷 구성:**
- `assets`: 공개 버킷 (프로필 이미지 등)
- `private`: 비공개 버킷

**파일 업로드 제한:**
- 최대 크기: 5MB
- 허용 형식: JPEG, PNG, GIF, WebP
- Presigned URL 만료: 10분
- 저장 경로: `profiles/{userId}/{UUID}.{extension}`
